# LocalChatApp 启动指南

**注意**：以下命令均假设您的命令行当前所在目录为项目根目录：
`d:\Code\doing_exercises\programs\LocalChatApp`

## 1. 编译项目

如果这是第一次运行或代码有过修改，请先执行编译：

```cmd
mvn clean compile
```

项目依赖已预先放置在 `lib` 目录下，启动脚本将自动加载。
（无需再执行 `mvn dependency:build-classpath`）

## 2. 启动服务端

在终端中执行以下命令（推荐使用脚本）：

```cmd
.\run_server.bat
```

或者手动执行：

```cmd
java -Djava.net.preferIPv4Stack=true -cp "target\classes;lib\*" server.Server
```

**关闭服务端的方法**：
在服务端运行的终端窗口中输入 `exit` 然后按回车键，服务器将会保存数据并安全退出。

> 注意：直接关闭窗口可能会导致部分数据丢失。

## 3. 启动客户端

请打开一个新的终端窗口（可打开多个以模拟多用户），执行以下命令（推荐使用脚本）：

```cmd
.\run_client.bat
```

脚本会自动配置依赖和 IPv4 栈。

或者手动执行：

```cmd
java -Djava.net.preferIPv4Stack=true -cp "target\classes;lib\*" client.App
```

## 4. 常见问题

- **Socket 错误**: 如果出现 `Invalid argument: socket`，通常是因为 IPv6 问题。启动脚本已添加 `-Djava.net.preferIPv4Stack=true` 参数来强制使用 IPv4。
- **JavaFX 缺失**: 确保 `lib` 文件夹下包含 JavaFX 的相关 jar 包（如 javafx-controls, javafx-fxml, javafx-base, javafx-graphics 等）。
